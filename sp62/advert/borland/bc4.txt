

  Новинка: Borland C++ 4.0

Power

Made

Easy

Компания Borland выпускает новое поколе-ние 32-разрядных
компиляторов для DOS, Windows 3.1, Win32s и Windows NT.

Расширения и изменения, внесенные в новую версию, повышают
производительность труда программиста благодаря полностью
настраиваемой, открытой Интегрированной Среде Разработки (ИСР),
переработанной библиотеке объектов и новым возможностям
компилятора, способного генерировать 16-ти и 32-разрядные
приложения и поддерживающим последние рекомендации комитета ANSI
C++. Важные нововведения - средства визуального
программирования, профессиональный настраиваемый редактор,
обладающий возможностями лучшего редактора для программистов
BRIEFй, Turbo Debuggerй, ставший ещў мощнее, встроенная
поддержка формата VBX для С и С++ и многое другое.

Полная реализация ANSI C++ стандарта. Borland C++ 4.0
поддерживает последние спецификации ANSI-стандарта по языку C++:

	обработка исключительных ситуаций (exception handling) для С и
С++;

	поддержка информации о типе объекта на этапе исполнения (
Run-Time Type Information - RTTI );

	реализация класса стан-дартных  ANSI-строк;

	поддержка трўх символь-ных типов переменных.

  

Визуализация - успешное начало. Как и предыдущие версии
компилятора, новая версия показывает, что Borland по прежнему
намерен занимать ведущее место на рынке компиляторов C++.
Генератор приложений AppExpert поможет построить  большую часть
программы. Новичок имеет возможность сразу  получить развитое
MDI-приложение с трехмерными диалоговыми управляющими
элементами, работой с печатью и встроенной справочной системой.
Трехпанельный редактор ClassExpert, работающий в стиле
SmallTalk, поможет развить приложение, сгенерированное
AppExpert, породить классы, модифицировать методы. При этом в
соответствующих панелях доступны редактируемые классы,
функции-члены этих классов и тексты реализации выбранных функций.

16-ти и 32-разрядные инструменты в одной упаковке. Borland C++
4.0 включает и 16-, и 32-разрядные компиляторы, компоновщики,
отладчики и библиотеки, таким образом не возникает необходимости
покупать что-либо ещў для ведения разработок для Win32s или
Windows NT. Имея библиотеку Win32s, уже сейчас можно запустить
32-разрядные приложения в 16-разрядной оболочке  Windows 3.1.
Win32s представляет из себя специальный ретранслирующий уровень
над Windows 3.1, реализующий широкое подмножество API Windows
NT. Не устанавливая Windows NT, можно воспользоваться
преимуществами ■плоской■ модели памяти, обеспечиваемой
32-разрядным API Win32s.

32-разрядный вариант компилятора выполняет оптимизирующие
преобразования, похожие на их 16-разрядный аналог, типа выноса
инвариантного кода, оптимизации циклов и переходов, отслеживания
избыточных загрузок, глобального распределении регистров. Также
добавлены новые 32-разрядные методы оптимизации - планирование
регистров, объединение в пары и выборки для процессоров 486 и
Pentiumй.

Интегрированная Среда Разработки Borland C++ 4.0 работает под
Windows 3.1. ИСР теперь обладает редактором со встроенной
технологией BRIEF, что позволяет проводить многопанельное
редактирование и выделение столбцовых блоков. Из среды
разработки полностью контролируется компилятор, компоновщик,
опции библиотек с возможностью создания 16-ти и 32-разрядных
исполняемых файлов или DLL. Использование TargetExpert
значительно упрощает процесс взаимосвязи компонентов. Можно
перейти с 16-разрядного приложения на 32-разрядный вариант лишь
несколькими нажатиями кнопки мыши в диалоговом окне
TargetExpert, тем самым заменив все соответствующие опции
компилятора для всего проекта. TargetExpert позволяет создавать
в том числе и DOS-приложения.

Интегрированный отладчик TurboDebugger теперь может проводить
32-разрядную отладку, обладая уникальной способностью работать с
32-разрядными приложениями в среде Windows 3.1 при помощи
подмножества Win32s. TurboDebugger обеспечивает аппаратную
отладку, условные точки останова, поддержку обработки
исключительных ситуаций и возможность удалўнной отладки. Также
поддерживается отладка 16-разрядных Windows - приложений.

ProjectManager позволяет использовать заранее заданные наборы
стилей, для определения уровня оптимизации или для задания
отладочной информации. Можно определить уникальный стиль для
развиваемого проекта. При использовании визуального
ProjectManager есть возможность создания нескольких версий
проекта (к примеру, 16-ти и 32-разрядной) без необходимости
прибегать к помощи Make-файлов. Для этого достаточно поместить
общие файлы в ■пул исходных файлов■ (■source pool■), и
предоставить заботу об остальных деталях перестройки исполняемых
файлов ProjectManager.

Полный набор инструментов. Borland C++ 4.0 включает все
необходимые средства для ведения разработок как 16-ти, так и
32-разрядных программ. Редактор ресурсов Resource WorkShop
теперь поддерживает форматы 32-разрядных ресурсов, воспринимает
форматы Microsoft Visual Basic VBX и встроен в Интегрированную
Среду Разработки.

ObjectWindows 2.0: мобильная библиотека классов. Borland C++ 4.0
включает новую версию ObjectWindows - 2.0. Это библиотека
объектов высокого уровня, которые обеспечивают действительный
выигрыш от объектно-ориентированного программирования на Си++.
ObjectWindows расширена высокоуровневыми классами: Полоски
быстрого доступа (speedbar), статус-строки, классы для
управления документами и печатью, для работы с форматами VBX и
классы, покрывающие важную часть программирования в Windows -
работу с объектами Интерфейса Графических Устройств (GDI).

При переходе на 32-разрядные приложения Windows NT или Win32s
изменяются интерфейс программирования и протоколы сообщений.
ObjectWindows 2.0 скрывает различия между 16-ти и 32-разрядными
версиями интерфейсов. Можно написать код для одного окружения,
скажем Windows 3.1, и он будет работать без изменений  исходного
текста программы на всех Windows-платформах, включая Win32s и
NT. Так как библиотека обладает встроенной переносимостью,
программисту нет необходимости загромождать программу условными
конструкциями ■ifdef■, рассчитывая на разные версии Windows.

ObjectWindows 2.0, по сравнению с версией 1.0, переписана с
самого основания с использованием последних нововведений в Си++,
таких как  шаблоны и обработка исключительных ситуаций. Это
делает программы, использующие ObjectWindows, более ясными и
легче переносимыми на другие компиляторы. Таким образом,
программы, написанные с использованием ObjectWindows, могут быть
легко перетранслированы другими NT-компиляторами (DEC Alpha, SGI
MIPS). Обьекты ObjectWindows 2.0 также обладают встроенной
диагностикой и строгой проверкой типов, что существенно
облегчает процесс отладки.

Borland и Novell 4 октября 1993 года заключили соглашение о
совместной разработке и взаимном лицензировании ObjectWindows
для AppWare, мобильной библиотеки Novell. Новая версия
ObjectWindows будет обеспечивать переносимость программ между
Windows, Mac, OS/2 и UNIX, включая UnixWare, SunOS, HP-UX.
Последующие версии будут поддерживать Solaris, OSF/1 и AIX. 

При программировании на Borland C++ 4.0 не возникнет
необходимости изучать другие интерфейсы. Функционально
законченная библиотека ObjectWindows 2.0 обеспечит разработчика
необходимыми средствами для построения приложения, а AppWare
сделает так, чтобы программа заработала на любой из
поддерживаемых платформ. Текст программы, разработанной с
использованием библиотеки ObjectWindows 2.0 сегодня, завтра
может быть перенесўн во все основные графические окружения.

Возможно, Вы не знаете, в какой операционной системе Вы будете
работать в будущем. Однако, используя Borland C++ 4.0, Вы
получаете высокую производительность сегодня и переносимость
приложений завтра.
