TITLE: 950201 GPF IN KRNL386.EXE AT 0001:3A0D WHEN STARTING MS ACCESS.
 
DESCRIPTION OF SYMPTOMS:
    When MS Access is started from a full screen or seamless session,
    users receive a GPF in KRN386.EXE at 0001:3A0D.
 
SUMMARY OF RESOLUTION:
    Change the following settings in the MS Access object.
 
    DOS_BACKGROUND_EXECUTION=ON
    EMS_MEMORY_LIMIT=0
    XMS_MEMORY_LIMIT=2048
    DPMI_MEMORY_LIMIT=64
    DPMI_DOS_API=ON
    HW_ROM_TO_RAM=ON
    IDLE_SECONDS=60
    IDLE_SENSITIVITY=100
    DOS_HIGH=ON
    DOS_UMB=ON
    VIDEO_RETRACE_EMULATION=OFF
 
DETAILS OF RESOLUTION:
 
    1. In the WIN-OS/2 settings, with settings listed in left
       box, select the Help push button.
    2. Scroll to Related Information.
 
    3. Select the following DOS and WIN-OS/2 Settings:
 
 
    DOS_BACKGROUND_EXECUTION=ON
    EMS_MEMORY_LIMIT=0
    XMS_MEMORY_LIMIT=2048
    DPMI_MEMORY_LIMIT=64
    DPMI_DOS_API=ON
    HW_ROM_TO_RAM=ON
    IDLE_SECONDS=60
    IDLE_SENSITIVITY=100
    DOS_HIGH=ON
    DOS_UMB=ON
    VIDEO_RETRACE_EMULATION=OFF
 
    4.  After these are set, select the Save radio button.
    5.  Exit from program's settings.
