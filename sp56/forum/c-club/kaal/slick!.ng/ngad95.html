   When the new window is created with makewnd(), the new structure
 will be added to linked list of windows. This structure has following
 fields :

  o Frame corner coordinates. These are the maximum coordinates for
    viewport, which may change. Corner coordinates are used later to
    restore the screen contents, so try not to screw them up.
  o Viewport corner coordinates. They are used for outputting, scrolling
    and clearing the window and to position cursor inside the window.
    You may change them if you need, but avoid going over the limits
    defined with frame corners. These coordinates are normally set
    so, that frame will be saved on output.
  o Saved cursor position and size. They will be restored, when window
    is closed, so you can fool around with cursor any way you want.
  o Colour used to open this window. Could be useful when you want to
    output something to your window.
  o Two pointers pointing to next and previous structure. They are the
    only link in this world, if you open 3 windows, using pushwnd() twice
    and then use closewnd(), all your windows will be gone forever!
  o Current cursor position. Used by putcwnd() when emulating teletype.
  o Cursor position before push.

 Here is how it looks in SLICK!.H
 ───────────────────────────────────────────────────────────────────────────
 typedef struct    {
     int x1,y1,x2,y2;         /* frame corners */
     int vx1,vy1,vx2,vy2;     /* viewport corners */
     unsigned int curpos,     /* cursor pos & size */
                 cursize;     /* before opening ... */
     unsigned int color;      /* used to open */
     void far *last;          /* to previous window */
     void far *next;          /* to next window */
     unsigned int cursor;     /* current cursor position */
     unsigned int pushcursor; /* used by pushwnd()/popwnd() */
     int space[1];            /* data, expands ... */
 } WINDOW;
