   This is structure for video control block, mostly for package internal
 use, but you'll find some useful information in structure defined in
 SLICK!.H as

   extern VIDEO far _videocblk;

 Here is structure itself:
 ───────────────────────────────────────────────────────────────────────────
 typedef struct {
    unsigned int base;        /* physical/virtual screen segment */
    char mode;                /* physical video mode */
    char cols;                /* columns on physical/virtual screen */
    char rows;                /* rows on physical screen */
    char snow;                /* bit0=1 forces snow check if colour mode */
    char ega;                 /* 0 - no EGA, 1 - EGA color, 2 - EGA mono */
 } VIDEO;
 ───────────────────────────────────────────────────────────────────────────

   This information will become valid after first call to low level drivers
 and will be refreshed by low level drivers if physical video mode changes
 or base is set to 0. You can define your own virtual screens by setting
 base to segment of your screen and filling up another fields. Note that
 low level drivers have no intrest to 'ega' and 'rows' fields, they will
 be set just for you...
