 Ivents signal system

     Operators at there work need some events signal like to confirm
     validity or alarm error situation. Original Clipper get system signal
     error situation with message at scoreboard (if SCROEBOARD ON) and
     if SET BELL ON and SET CONFIRM OFF signal last letter of valid field
     with fixed bell sound - TONE(500,9). CTK distinguish additional events
     and all of them has corresponding sound type. Audible signal
     prefirable at many cases. You can change friquency and duration of all
     sound type or tern then off. See xSet().

 Cursor

     In CTK get system cursor have Insert, Overwrite and Off state.
     If cursor is off (in Off state) then xSet[xCURSOR] == .F.
     Each state have its shape. Default shapes defined in CTK.CH.

     CTK get system sounds
     ────────────────────────────────────────────────────────────────
     InitCTK()                         Comment
     ────────────────────────────────────────────────────────────────
     xSet ({xCURSOR},      .F.)        initial cursor state is off
     xSet ({xINS_CURSOR},  SC_INSERT)  cursor shape for insert mode
     xSet ({xOVR_CURSOR},  SC_NORMAL)  cursor shape for overwrite mode
     xSet ({xNO_CURSOR},   SC_NONE)    cursor shape when cursor is off
     ────────────────────────────────────────────────────────────────

     If cursor is off when Read command executed then beafor its execution
     cursor turned on accordin to insert/overwrite state. On Read exit
     cursor returned to shape which was beafor Read.
