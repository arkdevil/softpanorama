 xSet()
 Change system setting with storing new and old setting in specified list
 ────────────────────────────────────────────────────────────────────────────────────

 Syntax

     xSet (aIndex, [xNewValue], [aSetList]) --> xOldValue

 Arguments

     <aIndex>
          is list of indexis to CTK or system setting

     <xNewValue>
          is an optional arguments which specifies a new
          value for the specified setting.

     <aSetList>
          optional list to store old and new set value. Later
          can be used by SetList.

 Returns

     xSet() return previous value of the setting.

 Description

     Function xSet() is extension of standard function Set().
     This function used to inspect or set Clipper system sets and CTK sets.
     Additionaly it store new and old set with there aIndex in
     aSetList if it's specified.

     The setting to be inspected or changed is specified using manifest
     constants in aIndex list. These constants are defined in header
     files SET.CH and CTK.CH. The header file must be included in top
     of source wich uses xSet. For each SET() there is corresponding
     form of xSet:

          SET(nSetting, xNewValue)  <==>  xSet({nSetting}, xNewValue)

     Also xSet() used to inspect/change CTK settings. All settings
     (standard and CTK) stored in public array (list) xSet. During
     InitCTK() evaluation this public initiated by all standard Clipper
     setting and default CTK setting. In difference to SET() scalar
     index,  xSet() use index combined from more then one number in
     list aIndex. For example CTK setting storing default Menu()
     frame color have index - {xCOLOR, xMENU, xFRAME}. With
     this piculiarity, settings can be arranged in group's (like xCOLOR,
     xSOUND groups). Index manifest constants for CTK sets defined
     in CTK.CH.

     Additionally xSet can handle Clipper sets which SET() can't and
     accessed by special commands (SetKey() etc).

     When you call xSet() and defined by aIndex dimension doesn't exist
     in public xSet then this dimension is created. If some index in
     aIndex greater then corresponding dimension then it is expanded
     to this index. This fiture allows to store application
     sets in public xSet.

     A aSetList used for storing old and new set value in so named
     Set List. Function xSet() called with a aSetList form Set List
     item structure from aIndex, xNewValue and current set value, and adds
     this item to Set List end.

     Set List item structure
     ────────────────────────────────────────────────────────────────
     CTK.CH        Value      Comment
     ────────────────────────────────────────────────────────────────
     SL_INDEX      1          aIndex value
     SL_NEW        2          xNewValue value
     SL_OLD        3          current (old) value of set
     ────────────────────────────────────────────────────────────────

     For speed reason you can replace xSet() call with direct access to
     public xSet by using following scheme:

          xSet ({N1,N2,...})   <==>  xSet [N1,N2,...]
