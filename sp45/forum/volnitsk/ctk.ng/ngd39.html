 InitCTK ()
 Initiate CTK environ and save DOS screen
 ──────────────────────────────────────────────────────────────────────────────────

 Syntax

     InitCTK ([nLangScheme, nLangToggleKey]) --> nil

 Arguments

     <nLangScheme>
          Optinal language scheme number. Languge scheme define keyboard
          layout, logic yes/no keys, date format and messages displayed at
          scoreboard.

     <nLangToggleKey>
          Inkey code wich will activate Languge Toggle Block.

 Returns

     InitCTK () always returns NIL

 Description

     Must be first statment in procedure main if you using CTK.
     Used in pare with EOJ(). Initiate CTK public varible xSet,
     save DOS screen, clear it and make such system sets:

          set  cursor      off
          set  scoreboard  off

     If language scheme specified then get system configured to support
     this languge with corresponding keyboard layout, language
     specific yes/no keys and date format. Currently available
     language schemes are:

     Defined languge schemes
     ────────────────────────────────────────────────────────────────
     CTK.CH                   Value     Comment
     ────────────────────────────────────────────────────────────────
     -                        NIL       Not use natinal languge support
     LS_SYSTEM                0         Not use natinal languge support
     LS_ENGLISH               1         Support for extended english languge
     LS_RUSSIAN               2         Support for russian languge
     LS_UKRANIAN              3         Support for ukranian languge
     LS_UKRANIAN_RUSSIAN      4         Support for ukranian as basic
                                        and russian as additional languge.
     ────────────────────────────────────────────────────────────────

     If you need language with not defined scheme use functions xSet
     to customize get system.

     For all sheme > 0 defined  default Languge Toggle Block which
     change current language on nLangToggleKey press according to
     following rule:

     <Language No> := <Language No> % <Number of Languages> + 1

     If here is only 2 languages then change sequence is: 1, 2, 1, 2 ...
