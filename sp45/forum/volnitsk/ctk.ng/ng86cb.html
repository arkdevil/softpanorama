     In CTK environ you can create any number of overlaping windows by
     calling SetWindow(). Each created window store its information
     in special Window Structure. It store info about window coordinates,
     frame type, title, used screen area and so on. On window creation
     this structure added to so named Window List internal to window
     system. On deletion of window by UnSetwindow() the last added
     Window Structure deleted from Window List. If you need to update
     not last (not active) window it can be overlaped by more recent window.
     In this case we must use GoToWindow().


     Window Structure
          Store information about window coordinates (with frame), window
          shadow coordinates, and char string for saving used screen
          area (shadow use screen too).

          Window Structure
          ────────────────────────────────────────────────────────────────
          CTK.CH         Value     Comment
          ────────────────────────────────────────────────────────────────
          TOP            1         top window margin coordinate
          LEFT           2         left window margin coordinate
          BOTTOM         3         bottom window margin coordinate
          RIGHT          4         right window margin coordinate
          SAVE_SCREEN    5         string storing screen area under window
          TOP_SHADOW     6         top shadow margin
          LEFT_SHADOW    7
          RIGHT_SHADOW   8
          BOTTOM_SHADOW  9
          W_MAX_WINDOW   9         window structure dimensions
          ────────────────────────────────────────────────────────────────
