<Краткая информация по Архиватору LHA>
 Рабочая версия - v2.13, 20/07-1991
 Создан Харуясу Йошизаки (Public Domain, 1988).
 Является  усовершенствованной  версией  программы
<LHARC> 1.13c (LH 1.13). Старый формат архива .LZH,
используемый 1.13c,  полностью  поддерживается, но
нет обратной совместимости.
 Является общепризнанным стандартом в Японии.
 В одной программе объединены все функции, включая
образование саморазворачивающихся архивов.
 Создает архивы с расширением .<LZH>.
 Файл в кодах ASCII <текстовый>  размером  500 kBt
упаковывает => на 88%  за 1 м. 50 с.
 Файл в машинных кодах тех же размеров упаковывает
           => на 58%  за 1 м. 15с.
 Недостатки: низкая  скорость работы  и отсутствие
защиты архивных файлов с помощью пароля.
 Рекомендуем для упаковки большого количества малых
файлов и при упаковке плохо сжимаемых файлов.

<Краткая информация по Архиватору PKZIP>
 Рабочая версия - v2.04, 25/01-1993
 Поставляется вместе с <PKUNZIP>. Продукт фирмы PKWARE
 <PKZIP> является инструментом для образования архивов
и исправления в них ошибок.
 Упаковывает файлы и структуры каталогов (даже пустые)
 <PKUNZIP> применяется для распаковки файлов.
 Создает архивы с расширением .<ZIP>.
 Файл в кодах ASCII <текстовый> размером 500 KBt
упаковывает: I  метод - на 85% за 452 сек;
             II метод - на 81% за 10(!) сек;
 Файл в машинных кодах тех же размеров сжимается:
             I  метод - на 42% за 35 сек;
             II метод - на 27% за 20 сек;
 Рекомендуется для упаковки выполняемых программ.
 По скорости распаковки - вне конкуренции !

<Краткая информация по Архиватору PKPAK>
 Рабочая версия - v3.61, 02/08-1988.
 Автор - Фил Кац. Продукт фирмы <PKWARE>.
 Является усовершенствованной  версией программы
PKARC (PKXARC) 3.5 (1987).
 <PKPAK>  является  инструментом для образования
архивов и исправления в них ошибок.
 <PKUNPAK> применяется для распаковки файлов.
 Создает архивы с расширением .<ARC>.
 В настоящее время эти программы не развиваются.
 Файл в кодах ASCII <текстовый> размером 500 kBt
упаковывает => на 84% за 10(!) сек.
 Файл в машинных кодах тех же размеров в среднем
сжимается   => на 26% за 20 сек.
 Работу со структурами каталогов не поддерживает.
 Рекомендуем для упаковки текстовых файлов !

<Краткая информация по Архиватору PAK>
 Рабочая версия - v2.51, 1988-90
 Продукт фирмы NоGATE CONSULTING, 1988 г.
 По всем параметрам занимает промежуточное место.
 Выполняет почти все действия, связанные с архивацией
 Максимальную степень уплотнения информации достигает
за счет уменьшения быстродействия.
 Архивы имеют расширение .<PAK>.
 Файл в кодах ASCII <текстовый> размером 500 kBt упа-
ковывает  => на 75% за 25(!) сек.
 Файл в машинных кодах тех же размеров упаковывает
         => на 35% за 55 сек.
 Рекомендуется для упаковки с паролем архива, создан-
ного ранее при помощи <LHICE> или <LHA>.

<Краткая информация по Архиватору ARJ>
 Рабочая версия - v2.30, 19/01-92
 Автор - Роберт К.Янг (1990г).
 Продукт фирмы Public Domain. Архивы имеют расширение .<ARJ>.
 Упаковывает файлы и структуры каталогов.
 Предлагает пять методов уплотнения: от 0-го(вообще не сжи-
мает файлов) до 4-го, сжимающего файлы с максимальной быст-
ротой (за счет ухудшения степени сжатия).
 Усовершенствованная защита от несанкционированного доступа
 При максимальной плотности упаковывает лучше чем <LHA>, но
несколько медленнее чем <PKZIP>.
 При ускоренном методе пакует быстрее и плотнее, чем PKZIP.
 Наиболее эффективно работает с файлами баз данных, тексто-
выми и файлами очень маленьких размеров.

<Краткая информация по Архиватору LIMIT>
 Рабочая версия - v1.0, ИЮЛЬ 1993
 Автор - J Y LIM (Ithaca, NY, USA).
 Архивы имеют расширение .<LIM>.
 Упаковывает файлы и структуры каталогов.
 Предлагаемый  метод  сжатия  основывается на использовании
словаря в окне, увеличивающемся до 32 кБайт плюс усовершен-
ствованный код Науфмана.
 Предлагает три метода уплотнения: от 0-го (вообще  не сжи-
мает файлов) до 3-го, сжимающего файлы с максимальной быст-
ротой (за счет ухудшения степени сжатия).
 Во всех режимах упаковывает лучше и быстрее чем архиваторы
<LHA>, <PKZIP> и <ARJ> !!
 Эффективно работает с файлами любых форматов и любых раз-
меров.
 Недостатки: остутствуют
 - защита от несанкционированного доступа;
 - режим <M>(ove) - перенос файлов в архив;
 - режим <F>(resh) - обновление файлов в архиве.
-----------------------------------------------------------
<Максимальное Уплотнение>
  В этом режиме архиватор пакует  файлы,  подбирая  для
каждого наиболее оптимальный метод сжатия. Это занимает
больше  времени,   зато  создается  архив  минимального
размера (для данного архиватора, разумеется).
  Рекомендуется  использовать эту функцию для  создания
долговременных архивов.
-----------------------------------------------------------
<Максимальная Скорость>
  Данный архиватор обладает возможностью сжимать файлы
ускоренным методом за счет ухудшения плотности паковки
  Время, затраченное на процесс упаковки,  существенно
снижается, но размер Архива при этом увеличивается.
  Рекомендуется использовать эту функцию лишь в случае
создания кратковременных архивов (например - для пере-
мещения файлов при помощи дискет).
-----------------------------------------------------------
<Без Сжатия (подклеить)>
  В этом режиме архиватор собирает файлы в один архив
за минимально короткое время, без упаковки, только
"подрезая" хвосты обрабатываемых файлов.
  Рекомендуется использовать эту функцию для склеивания
воедино одновременно нескольких ранее созданных Архивов
или плохо сжимаемых файлов.
-----------------------------------------------------------
<Сделать Архив Self-Extr>
  Благодаря этой функции Вы можете создавать <самораз-
ворачивающиеся> Архивы.  Созданный Архив  будет  иметь
расширение <.EXE> или <.COM>,  что позволит развернуть
его даже на таком компьютере, который не имеет програ-
ммы для распаковки.
-----------------------------------------------------------
<Добавить Комментарии>
  Имеется возможность снабдить весь архив в целом или
каждый  упаковываемый  файл в отдельности специальным
<текстовым комментарием>.
  Рекомендуем  применять  эту функцию  при   создании
долговременных Архивов.
-----------------------------------------------------------
<Включая подкаталоги>
  При  вызове этой функции программа-архиватор создает
Архив из  выбранных файлов текущего каталога, а  также
из  всех <подкаталогов>, выходящих из текущего. Удобно
использовать  эту  функцию для 100%-го воспроизведения
структуры дерева каталогов.
-----------------------------------------------------------
<Защита Архива Паролем>
  Благодаря этой функции  Вы можете  создавать Архивы с
засекреченной информацией.  Необходимо  ввести ключевое
слово <пароль> - и распаковать такой  Архив  не удастся
никому из посторонних.  Однако  не следует  применять в
виде пароля свое Имя или Дату рождения!
  Вы даже не представляете,  какую массу архивов  можно
"вскрыть", зная только имя его создателя!
  В программе используются  специальные символы, позво-
ляющие архиваторам  PKZIP и ARJ  запрашивать Пароль при
запуске с соблюдением полной секретности ввода символов
ключевого слова ( см. Помощь на [Ввод Пароля] ).
  Данную Функцию можно также вызвать, нажав клавишу [P]
при активной Панели Архива.
-----------------------------------------------------------
<Много-Томный Архив>
  Данный архиватор обладает возможностью создать  один
Архив на нескольких дискетах <томах>. После заполнения
свободного  пространства на  одной  дискете  архиватор
делает паузу и предлагает сменить диск в дисководе.
  Функция весьма удобна для упаковки больших файлов, а
также при создании резервных копий Винчестера.
-----------------------------------------------------------
<Разархивация>
 Команда для <распаковки всего Архива>.
 При распаковке восстанавливается первоначальная <структура
дерева> каталогов.
 Если  обрабатываемый  Архив  был  создан  с <Паролем>,  то
нажмите ключ [P] и введите Пароль (без ошибок!)
 Если Вам надо распаковать отдельные файлы,  воспользуйтесь
ключами [F5]Копия или [F6]Перенос.
 Перед  <Распаковкой>   будет  проверен  размер  незанятого
пространства  на текущем  Диске  и  выдано  сообщение, если
его недостаточно для размещения новых файлов!
 Данную Функцию можно также вызвать,  нажав клавишу [E] при
активной Панели Архива.
----------------------------------------------------------
<Тестирование>
 Команда для проверки <целостности> Архива.
 Подсчитывается <Циклическая Контрольная Сумма>  каждого
архивного файла и сравнивается с тем значением,  которое
хранится в Архиве, с выдачей сообщения о результате.
 Если  обрабатываемый  Архив был создан с <Паролем>,  то
нажмите ключ [P] и введите Пароль (без ошибок!).
 Если Вы пометили один файл или  целую  группу   файлов,
команда  будет выполняться именно для них,  в  противном
случае тестируется файл или каталог, выделенный курсором
 Данную Функцию можно также вызвать,  нажав  клавишу [T]
при активной Панели Архива.
----------------------------------------------------------
<Замена Старого>
 Команда для <обновления Архива>.
 Выполняется для всего Архива целиком.
 Бывает, что со временем  некоторые  файлы  подвергаются
изменениям,  и Вам  захочется обновить их  старые Копии,
хранящиеся внутри Архива. В этом случае данная процедура
может оказаться весьма полезной.
 Только  постарайтесь правильно  указать рабочий Каталог
для поиска обновленных файлов.
 Данную Функцию можно также вызвать,  нажав клавишу  [F]
при активной Панели Архива.
----------------------------------------------------------
<Добавка Нового>
 Команда для <обновления> и <увеличения> содержимого Архива.
 Выполняется  аналогично  команде - <Замена Старого>, но при
этом  добавляет  в  Архив  из  выбранных Каталогов те файлы,
которых в нем нет.
 Внимание !!!  Если Вы  неверно  указали рабочий Каталог, то
рискуете заполнить Архив ненужной информацией. В этом случае
воспользуйтесь командой  [F8]Стереть  и впредь  будте  более
внимательнее!
  Выполняется для всего Архива целиком.
  Данную Функцию можно также вызвать,  нажав клавишу [U] при
активной Панели Архива.
----------------------------------------------------------
<Ввести Пароль>
  Используйте эту Функцию для работы  с Архивами,  данные в
которых были засекречены при помощи  <Пароля>.  В программе
используются специальные символы,  позволяющие  архиваторам
PKZIP  и  ARJ  запрашивать Пароль при запуске с соблюдением
полной секретности ввода символов ключевого слова (так, для
Архиватора <ARJ> это символ [?],  для <PKZIP> - [Пробел].
  Данную Функцию можно также вызвать, нажав клавишу [P] при
активной Панели Архива.
----------------------------------------------------------
<ПереИменовать>
  Используйте эту Функцию для переименовки рабочего Архива.
Это рекомендуется делать сразу,  если Вы начинаете работу с
новым Архивом ( иначе  ему будет присвоено  стандартное имя
<NONAME> ).
  Данную Функцию можно также вызвать, нажав клавишу [N] при
активной Панели Архива.
----------------------------------------------------------
<Новый Архив>
  Используйте эту Функцию,  если хотите упаковывать файлы в
новый Архив.  При вызове  Вам будет  предложен  список-Меню
пяти известных Архиваторов,  на работу с которыми  ориенти-
рован <Архивный Процессор>.
  Важно!  Местоположение нового  Архива сразу привязывается
к текущему каталогу Диска.  Если  выбранный  каталог Вас не
устраивает, перейдите в другой и повторите вызов Функции.
  Данную Функцию можно также вызвать, нажав клавишу [A] при
активной Панели Архива.
----------------------------------------------------------
<Краткий Режим>
  В этом режиме на панель выводятся только  имена файлов.
Вывод осуществляется в три колонки,  что  позволяет сразу
просматривать до 42 файлов одновременно. Панель имеет вид
       <╔══════════ 1_DIR\2_DIR\3_DIR ═════════╗

       ║2_file   ext│17_file  ext│31_file  ext║

       ║14_file  ext│28_file  ext│42_file  ext║
       ╟────────────┴────────────┴────────────╢
       ║..           <UP--DIR> 23/02-93  15:55║
       ╚══════════════════════════════════════╝>
----------------------------------------------------------
<Полный  Режим>
  В этом режиме на панель выводится список файлов с указанием
Размера каждого файла, Даты его создания и, для Панели Архива
 - Сжатого размера, для Панели Диска - Времени создания.
         <╔══════════ 1_DIR\2_DIR\3_DIR ═════════╗

         ║2_file   ext│     2183│ 6/05-93│ 13:22║

         ║    . . .   │  . . .  │  . . . │  ... ║
         ║14_file  ext│    48959│12/04-93│ 13:52║
         ╟────────────┴─────────┴────────┴──────╢
         ║..           <UP--DIR> 23/02-93  15:55║
         ╚══════════════════════════════════════╝>
----------------------------------------------------------
<Имена Файлов>
  При выборе этого пункта список файлов активной Панели
будет пересортирован по <Именам> файлов, от A до Z (или
от А до Я для русифицированных имен).
----------------------------------------------------------
<Расширения>
  При выборе этого пункта список файлов активной Панели
будет пересортирован по <Расширениям> файлов, от A до Z
(или от А до Я для русифицированных имен).
----------------------------------------------------------
<Размер Файлов>
  При выборе этого пункта список файлов активной Панели
будет пересортирован по  <Размерам>  файлов.  Первым  в
списке будет файл с минимальным размером, последним - с
максимальным.
----------------------------------------------------------
<Даты создания>
  При выборе этого пункта список файлов  активной  Панели
будет пересортирован по <Датам создания> файлов.  Первыми
в списке будут новые, недавно созданные файлы, последними
будут файлы более поздние по времени.
----------------------------------------------------------
<Время создания>
  При выборе этого  пункта  список файлов Дисковой Панели
будет пересортирован по <Времени создания> файлов.
Первыми в списке будут файлы, созданные в первой половине
дня (до 12 ч.),  последними - файлы,  созданные во второй
половине дня (после 12 ч.).
----------------------------------------------------------
<Cжaтый paзмep>
  При выборе этого пункта список файлов Архивной  Панели
будет пересортирован по <Сжатому размеру> файлов. Первым
в списке будет файл с минимальным размером, последним  -
с максимальным.
----------------------------------------------------------
<По каталогам>
  При выборе этого пункта список файлов Архивной  Панели
будет выводиться с учетом структуры дерева каталогов.
  При этом текущим станет каталог, в котором размещается
файл, выделенный курсором в настоящий момент.
----------------------------------------------------------
<Без каталогов>
  При выборе этого пункта список файлов Архивной  Панели
будет выводиться  без  учета структуры дерева каталогов,
т.е. одни файлы, без каталогов. Данный режим бывает поле-
зен, если Вам надо найти файл среди множества каталогов.
----------------------------------------------------------
<Koнфигуpaция>
  При выборе  этого пункта производится  сохранение  текущей
конфигурации <Архивного Процессора>. Сюда входит:
  - взаимное расположение Архивной и Дисковой <Панелей>;
  - текущий рабочий <Архиватор>, как выбираемый по умолчанию
при вызове программы без указания рабочего архива;
  - текущий <режим вывода> списка файлов отдельно для каждой
    из Панелей;
  - текущий режим <сортировки> файлов отдельно для каждой из
    Панелей.
  Важно!  При сохранении конфигурации осуществляется запись
установленных значений непосредственно в EXE-файл программы,
для чего требуется, чтобы файл AP.EXE  имел  первоначальный
размер, не сжатый утилитами типа DIET или PKLITE.

<Информация по сервису Упаковки>
  В случае, если Вы нажали ключ [F5], в рабочий Архив будут
помещены копии выбранных Вами файлов.  Если был нажат  ключ
[F6] - файлы будут перенесены с Диска в Архив.
  При этом будут учтены и те Опции,  которые  Вы пометили в
Меню-Архиваторы/Опции ( с ПодКаталогами, с Паролем и т.д.).
 [Продолжать] - выберите этот пункт,  и начнется выполнение
                выбранной команды упаковки.
 [Смена Каталога] - для изменения текущего  каталога Архива
 [Изменение Диска] - не работает,  т.к. в Архиве нет Дисков
 [Отказ] - выберите этот пункт если  передумали продолжать.
           Тот же эффект - клавиша [Esc].
----------------------------------------------------------
<Информация по сервису Распаковки>
  В случае, если Вы нажали ключ [F5], в текущий каталог на
Диске будут распакованы копии выбранных Вами файлов.  Если
был  нажат  ключ [F6] - файлы будут извлечены из Архива на
Диск (т.е. уничтожены в Архиве после распаковки).
  Если Вы выбрали команду Меню-Функций  [E]Распаковать, то
будет распакован весь Архив целиком независимо от текущего
каталога в Архиве.
  Если была выбрана команда Меню-Функций [F]Замена_Старого
или [U]Добавка_Нового, то обновление файлов будет произво-
диться по всему Архиву,  а поиск файлов на Диске - по всем
подкаталогам, выходящим из текущего ( количество указанных
в подсказке файлов может и не соответствовать реальному ).
 [Продолжать] - выберите этот пункт, и начнется выполнение
                выбранной команды распаковки.
 [Смена Каталога] - для изменения текущего каталога Диска.
Внимание!  Если на Вашем Диске доступна утилита 'NCD.EXE',
будет произведен ее автоматический вызов.  Это  лучше, чем
самому вводить с клавиатуры длинные маршруты Пути.
 [Изменение Диска] - аналогично функциям клавиш <Alt-F1/F2>
 [Отказ] - выберите этот пункт если передумали продолжать.
           Тот же эффект - клавиша [Esc].
----------------------------------------------------------
<Общая Информация по действующим ключам>
[Enter] - смена каталога; смена рабочего Архива (по расширению файла).
 Пометка отдельных Файлов - [Ins] или Правый[+], отмена - Правый[-].
 Смена текущего Диска - <Alt-F1> или <Alt-F2>.
[F1] - получение краткой справки по режимам или пунктам активных Меню;
[F2] - сервис для файлов активной Панели (сортировка, режимы вывода),
       переход к Меню Архиваторов или Меню рабочих Функций для Архива;
[F3] - просмотра содержимого файла (на Диске или в Архиве);
[F5] - упаковка файлов, если активна Панель Диска и распаковка файлов,
       если активна Панель Архивов;
[F6] - аналогично [F5], но с уничтожением выбранных файлов;
[F7] - для создания новых каталогов. Важно! После повторного чтения
       Архива Пустые каталоги в нем не сохраняются;
[F8] - уничтожение файлов. Для большей безопасности уничтожаются лишь
       помеченные файлы и требуется подтверждение выбранной команды;
[F10]- завершение программы ( аналогично - клавиша [Esc] ).
 Распаковка всего архива- ключ [E]   ПереИменовать Архив - ключ [N]
 Тестирование файлов    - ключ [T]   Начать новый Архив  - ключ [A]
 Замена старых файлов   - ключ [F]   Убрать/Вернуть Панели   <Ctrl-O>
 Добавка новых файлов   - ключ [U]   Поменять Панели местами <Ctrl-U>
 Ввести Пароль          - ключ [P]   Краткая информация      <Ctrl-L>
