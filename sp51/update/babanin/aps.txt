                                1. Введение.



   В последние годы все большую популярность  приобретают  программы  упаковки
(уплотнения)  данных.  Это  связано с тем,  что современные программы упаковки
располагают возможностями, значительно выходящими за рамки обычного уплотнения
файлов, а именно: дают возможность создания "архивов", содержащих большое чис-
ло упакованных файлов.  В результате получается двойной эффект:  во-первых, на
диске  освобождается отведенное для них место,  во-вторых,  упаковка некоторой
программы с различными вспомогательными файлами в одном архиве значительно об-
легчает операции переноса на другую систему,  создание копий и т.д.  В связи с
этим программы упаковки-распаковки правомерно называть "архиваторами".  Управ-
ление  архиваторами  осуществляется путем использования командного языка,  что
при работе с архивами делает процесс скучным и трудоемким. Этот недостаток ар-
хиваторов  породил новую категорию программного обеспечения - оболочки архива-
торов. Из зарубежных средств такого класса наибольшее распространение получили
оболочки NARC ( разработка фирмы Infinity Design Concepts ) и SHEZ ( автор Jim
Derr ).
   Одной из первых отечественных разработок этого класса является оболочка APS
( Archive Programs Shell ). APS - это оболочка, предназначенная для управления
архиваторами, построенная в виде многооконной системы меню. При разработке APS
ставилась задача достичь максимальной простоты в работе с  программой.  Интер-
фейс  пользователя  обеспечивает  одинаково  хорошее управление как при помощи
"мыши", так и с клавиатуры. Имеется возможность выбора элементов из меню путем
набора подсвеченных символов.
   Одним из отличий APS от известных систем является возможность  самотестиро-
вания в момент запуска на зараженность файловыми вирусами,  что позволяет выя-
вить наличие активных вирусов в компьютере и не проводить работы  по  упаковке
данных, дабы не внести зараженные вирусами файлы в архив.

   Оболочка архивных  программ  APS  обеспечивает интерактивное взаимодействие
пользователей со следующими программами - архиваторами:

  ■ PKARC(v 3.5), PKPAK(v 3.61), PKZIP(v 1.02,1.1,1.93), PKWARE Incorporated;
  ■ PAK(v 1.6), NoGateConsulting;
  ■ LHARC(v 1.131,2.05), Haruyasu Yoshizaki;
  ■ ICE(v 1.14), Haruyasu Yoshizaki;
  ■ ARJ(v 1.00,2.2), Robert K Jung;
  ■ CHARC(v 1.03), СП "Диалог", C.Чернивецкий.

   Для функционирования APS необходимы:

            - MS DOS версий 3.2 и выше;
            - адаптеры VGA, EGA, CGA;
            - оперативная память не менее 512 кбт.

       Способ распространения - FreeWare.

                     2. Настройка и запуск APS.

   APS взаимодействует с программами-архиваторами как с сопроцессами,  исполь-
зуя следующие фиксированные имена файлов программ-архиваторов:

  для расширения .ZIP :    PKZIP.EXE, PKUNZIP.EXE;
  для расширения .ARC :    PKARC.COM, PKXARC.COM;
  для расширения .PAK :    PAK.EXE;
  для расширения .LZH :    LHA.EXE;
                 .ICE :    LHA.EXE;
  для расширения .CHZ :    CHARC.EXE;
  для расширения .ARJ :    ARJ.EXE

  В режиме настройки возможна коррекция таблицы имен архиваторов. Оболочка ар-
хиваторов APS и сами программы-архиваторы могут находиться в разных директори-
ях, на разных дисках; при этом достаточно лишь указать путь в PATCH для успеш-
ного их поиска.  Поиск  соответствующих  программ производится в следующем по-
рядке:

  - в текущем директории;
  - в директории исполнения программ DOS 3.XX;
  - в директориях, указанных в PATH.

Для создания рабочих наборов при просмотре файлов используется диск, являвший-
ся текущим при запуске системы APS.


                          2.1. Запуск APS.

   Запуск оболочки архиваторов осуществляется путем ввода строки:

                            APS.EXE или
                            APS.EXE <имя архивного файла>

   В процессе работы используются следующие клавиши:

         F1 - помощь
         F2 - справка по методам сжатия;
         F3 - переключение языка общения (русский, английский); 
         F5 - переключение  формата просмотра каталога архивного файла;
         F6 - переключение экранов( APS и архиваторов);
        F10 - справка по использованию функциональных клавиш;
        ESC - возврат на предыдущий уровень;
      ENTER - выполнение функции;
      SPACE - выделение файлов при групповых операциях;
      PgDn  - листание страницы вниз;
      PgUp  - листание страницы вверх;
      End   - переход к концу просматриваемого текста;
      Home  - переход к началу просматриваемого текста;
      Gray+ - выделение всех файлов;
      Gray- - отмена всех выделенных файлов;

   При работе с "мышью" назначение клавиш следующее:

            Left  - выполнение функции
            Right - возврат на предыдущий уровень
       Left+Right - помощь

   Выбор соответствующей команды осуществляется путем перемещения подсвеченной
строки с помощью клавиш управления курсором (или движением  "мыши"),  а  также
может быть выполнен при помощи нажатия выделенного символа в названии команды.
Клавиши, доступные в текущем режиме, отображаются на 23-й сроке экрана.
   В APS иммется возможность установить желаемые атрибуты цвета экрана,  скор-
ректировать таблицу имен архиваторов, установить режимы их работы. Вхождение в
режим настройки осуществляется при помощи функции SETUP.

                      2.2. Настройка атрибутов цвета экрана.

   Функция SETCOLORS позволяет установить атрибуты цвета, а именно (рис.3):

    Frame - установка цвета обрамления окна;
    Title - установка цвета заголовка;
     Body - установка цвета фона;
 Selected - установка цвета подсвеченной строки;
     Mark - установка цвета выделенной строки;
     Help - установка цвета текста помощи;
     Info - установка цвета информационных строк.

   Выбрав соответствующую функцию и нажав ENTER получаем доступ к таблице цве-
тов  (рис.3),  в которой при помощи клавиш управления курсором выбираем нужный
атрибут цвета.  Для сохранения выбранных атрибутов цвета необходимо  выполнить
функцию SaveSetUp. Атрибуты хранятся в файле APS.INI.

                2.4. Установка режимов работы архиваторов.


   Функция OPTIONS позволяет устанавливать  различные  опции  для  программ -

  ■ Move Files         - перемещать  отмеченные файлы в архив;
  ■ Add Files          - заносить в архив те файлы, которых там не было;
  ■ Freshen Files      - заносить в архив только измененные файлы;
  ■ UpDate Files       - заносить в архив все отмеченные файлы;');
  ■ Create/Save SubDir - сохранить поддиректории  в архиве при упаковке и
                         создать сохраненные директории  при  распаковке  (ис-
                         пользуется только для архивов типа ZIP,  LHARC,  ARJ,
                         CHARC);
  ■ Show all Files     - при переходе в режим распаковки отображаются все
                         файлы директория(Yes) или только файлы с текущим рас-
                         ширением(No).

        3. Функциональные возможности оболочки архиваторов APS.


   В APS реализованы следующие функции:

      - просмотр каталогов архивных файлов;
      - помещение файла (или группы файлов) в архив;
      - смена текущего диска;
      - смена текущего типа архиватора;
      - установка режимов работы;
      - установка/снятие пароля на доступ к архивному файлу;
      - извлечение файла (или группы файлов) из архива;
      - просмотр файлов в архиве;
      - распечатка файлов из архива;
      - удаление файлов из архива.

                3.7. Извлечение файлов из архива.

   Подсвеченный или отмеченные в каталоге файлы извлекаются из архива и  поме-
щаются на диск,  номер которого запрашивается у пользователя. В процессе реар-
хивации на экране можно наблюдать ход процесса и сообщения программ -  архива-
торов. По завершению процесса архивации возможен повторный просмотр   рабочего
экрана архиваторов путем использования клавиши F6.

                      3.8. Просмотр файлов.


Текущий файл(подсвеченный в каталоге) извлекается из архива и выводится на эк-
ран порциями по 21 строке(рис.12).  Клавиши PgDn,  Home,  End,  PgUp, Down, Up
позволяют легко просмотреть файл.  Для прекращения просмотра нужно нажать кла-
вишу ESC. В верхнем левом углу экрана отображается номер первой строки (Line
Num ).

                        3.9. Распечатка файлов.

"Printed String ?" нужно ввести номер начальной строки  и  нажать  на  клавишу
"Enter".  В процессе печати отображается номер текущей распечатываемой строки.

                   4. Сообщения об ошибках.

    При возникновении ситуаций,  когда дальнейшая работа невозможна APS выдает
сообщение об ошибках, а именно:

 File(s) FileName not found - файл(ы) FileName не найден(ы);
             Path not found - путь не найден;
         File access denied - ошибка обмена;
        Invalid file handle - непр.обработка файла;
        Insufficient memory - нет памяти;
         Invalid enviroment - непр.среда;
             Invalid format - непр.формат;
           Write protection - защита записи на диске;
                    No name - нет имени;
   Pushed Ctrl/Break button - нажата клавиша Ctrl-Break;
            Unpacking error - ошибка распаковки;
FileName: Wrong structure archive file - нераспознана структура архивного
                             файла;
          Printer not ready - нет готовности принтера;
  Too many files in archive - переполнен каталог архивного  файла;
           Driver not ready - нет готовности устройства;
              Dos Error nn  - ошибка DOS с номером nn.
