      Эта команда  предоставляет  Вам  меню опций,которые являются гло-
 бальны и управляют всей компиляцией,если только  они  не  игнорируются
 внутреннимим метаоператорами исходной программы.  Эти опции могут сох-
 раняться в файле конфигурации через меню Setup (установка).
      Compile to  (компиляция в) - эта опция позволит Вам компилировать
 программу в память,файл .EXE или .TBC.
      Компиляция в  память  является  наилучшим средством для отладки и
 тестирования программы в связи с наивысшим быстродействием. Компиляция
 в  .EXE  файл представляет собой создание на диске выполняемого из DOS
 файла с расширением .EXE.Обычно этот режим выбирается  после  того,как
 программа  была  отлажена  в Turbo-Basic,хотя некоторые характеристики
 языка могут быть протестированы лишь в DOS (например, CHAINING - сцеп-
 ление).  Компиляция  в  .TBC  файл  создает файл,лишенный библиотечных
 подпрограмм.  Поэтому этот файл пригоден только для сцепления или  вы-
 полнения из файла .EXE,сам же по себе он выполняться не может.
      Следующие 5 опций применяются для  отладки  программ,поэтому  при
 создании  .EXE файла отлаженной программы выключите эти опции,т.к. они
 будут занимать лишнее место в Вашей программе.
      8087 required  (требуется 8087) - применяется для повышения быст-
 родействия в операциях с плавающей запятой.Если это  опция включена,то
 в  состав  микро-ЭВМ должен входить сопроцессор 8087,если нет,то прог-
 рамма будет использовать системные возможности для этих операций (хотя
 и более медленно).
      Keyboard break (прерывание с клавиатуры) - при включении этой оп-
 ции Вы можете прервать выполнение программы по Ctrl-Break.Но программа
 не прерывается во время вывода текста на экран или принтер или  выпол-
 нения ввода с клавиатуры.
      Bounds (границы) - в включенном состоянии включает в процесс ком-
 пилирования  проверку нижнего индекса массива при его заполнении.  При
 включении этого контроля ошибка будет выдаваться всякий раз,когда ваша
 программа  попытается использовать слишком большой или слишком малень-
 кий нижний индекс для данного массива.Например,в программе

      DIM Array(100)
      x=114
      y=Array(x)

 хотя она и пройдет компиляцию,но во  время  прогона  будут  обнаружена
 ошибка.
      Overflow (переполнение) - переполнение происходит,когда результат
 арифметической  операции  превышает  емкость  хранения целых чисел (от
 -32.768 до 32.767).Эта ситуация показана в примере.Если Вы

      x%=22555
      x%=x%+19000

 компилируете эту  программы  при включении контроля переполнения,то X%
 присваивается ошибочное значение.  Используйте функции CSNC и CDBL для
 выполнения целочисленных операций в формате представления чисел с пла-
 вающей запятой.  Заметьте,что контроль переполнения происходит в отно-
 шении  целых  чисел,  длинных целых чисел или любых типов вычислений с
 плавающей запятой.
      Stack test  (тест  стека) - при активизации этой опции происходит
 контроль стека при вызове подпрограмм,функций или процедур.Если Вы по-
 дозреваете,что  Ваш  стек может истощиться при этих операциях,включите
 эту опцию.  Для зарезервирования большего пространства под стек  исполь-
 зуйте метаоператор $STACK.
      Parameter line (строка параметра) - побуждает  COMMANDS$  возвра-
 щаться  в прогоняемую программу,и таким образом Вы можете моделировать
 информацию строки команды при отладке в Turbo-Basic.
      Metastatements (метаоператоры)  -эта  опция  имеет в свою очередь
 три опции:
      Stack (стек) - используется для определения (или повторного опре-
 деления) размера стека.Умолчание составляет 1024 десятичных (400 шест-
 надцатеричных)  байт.При  выборе  этого параметра Вы можете установить
 глубину стека от 400 до 7FFE байт (16-х) (См.  метаоператор  $STACK  в
 Гл.5).
      Music (музыка) - определяет размер буфера фоновой музыки. Если Вы
 не  используете  SOUND или PLAY,то можете уменьшить размер Вашей прог-
 раммы за счет установки буфера на 0.Максимальный размер буфера состав-
 ляет  4096 нот (8 нот заключены в 1 байт).Умолчание составляет 32 ноты
 (256 байт) (см. $SOUND ).
      Communications (связь)  -  имеет  отношение к приемному буферу по
 каждому  порту   связи.Минимальное   значение   равно   0,максимальное
 32.767,умолчание - 256 байтов.Заметьте,что в самой программе Вы можете
 изменить этот параметр метаоператором $COM n.
