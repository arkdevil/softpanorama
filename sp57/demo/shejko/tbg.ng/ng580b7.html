         аявление CALL NTERRUPT использует номер вектора прерывания для полу-
   чения подпрограммы на языке ассемблера. тот вызов обычно  применяется  для
   вызова  обработчиков DOS и BIOS-прерываний, но может также применяться для
   вызова обработчиков программных прерываний, определяемых пользователем.
         CALL INTERRUPT имеет следующий синтаксис:
         CALL INTERRUPT <целое выражение >, где целое выражение должно приме-
   няться как значение в диапазоне от 0 до 255.
         Перелд осуществлением вызова с прерыванием, вы должны настроить  со-
   ответствующие  регистры, используя заявление REG. Результаты, возвращаемые
   в регистры, хранятся внутренне компилятором, а доступ к ним возможен  спо-
   мощью функции REG.
         После завершения желаемой функции процедура на языке ассемблера дол-
   жна выполнить RETURN (возврат) из прерывания (IRET) к Турбо-Бейсику. Затем
   исполнение продолжается с заявления, следующего за CALL.
         Программист ответственен за защиту регистров стекового сегмента (SS)
   и стекового указателя (SP)
         Примечание:  DOS-прерывания  25Н  и  26Н оставляют регисть флагов на
   стеке,что противоречит приемам программирования применительно к 8086.  За-
   явление  CALL  INTERRUPT  Турбо-Бейсик обрабатываетдля вас эту несовмести-
   мость, автоматически подстраивая стековый указатель. Прерывание 25Н -  это
   абсолютное чтение диска, а 26Н - абсолютная запись диска.
