 Синтаксис:   CALL procname[(parameter list)]

 Комментарии: procname  -  имя процедуры,определенной в программе с по-
              мощью оператора SUB.
              parameter list - необязательный,разделенный запятыми, пе-
              речень переменных,выражений,постоянных,котороые передают-
              ся в процедуру. Есть три типа процедур:STATIC,стандартные
              процедуры, INLINE.Компилятор обрабатывает список парамет-
              ров в зависимости от типа процедуры.  Число и тип пересы-
              лаемых в  процедуру  аргументов  должен  соотноситься  со
              списком параметров в определении procname Если тип проце-
              дуры INLINE,то число аргументов переменно и  в  отношении
              параметров  не  производится проверка типа.Сама процедура
              несет ответственность за определение числа и типов  пере-
              сылаемых  к  ней  параметров  (этот  тип используется для
              включения подпрограмм в объектных  кодах  (см.  подробнее
              Приложение C)).  Когда при обращении к процедуре применя-
              ется метод 'пересылка ссылкой',то процедура  может  изме-
              нять передаваемые ей переменные.Если переменные заключены
              в круглые скобки,  то они пересылаются значением и их на-
              чальные значения не могут быть изменены (см.  Гл.4 'Опре-
              деление процедур и  функций  и  ьод  выполнения  програм-
              мы).Заметьте,что    выражения    и    постоянные   всегда
              пересылаются значением.  Аргументы  массива  определяются
              использованием  круглых  скобок  после  имени массива.Все
              массивы пересылаются ссылкой,например:

              CALL SumArry (a())
              CALL SumArry (a(3))


 Пример:      DIM Array (1)
              SUB TestProcedure(I%,L&,S!,D#,E,A(1))
                PRINT I%,L&,S!,D#,E,A(0)
              END SUB
