         Когда вы пользуетесь микроассемблером, фактически  вы  нуждаетесь  в
   4-х программах: редакторе(подобном отрывному блокноту секретаря), микроас-
   семблере, связчике и программе, называемой EXE2BIN.
         Сперва  включайтесь в вашу программу на ассемблерном языке с помощью
   редактора. Далее для создания обьектного файла,  прогоняется  микроассемб-
   лер.  Затем используйте связчик (LINK на вашем DOS-диске или связчик, пос-
   тавляемый с вашим микроассемблером ) для  создания  исполнительного  (EXE)
   файла.
         Наконец применяется EXE2BIN (включение в ваш DOS-диск) Для преобра-
         зовапния исполнительного файла в COM-файл.
         Для автоматизации этой обработки Вы  можете  использовать  следующий
   пакетный файл.
        MASM %1;
        LINK %1;
        EXE 2BIN %1 %1 COM
        Ниже  даются некоторые направляющие указания для преобразования прог-
   раммы ассемблерного языка из EXE-файла в COM-файл, подлежащий  использова-
   нию Турбо-Бейсиком.
         Не  включайте  стековый  сегмент (связчик будет сообщать об ошибке -
   нет стекового сегмента, но это игнорируется).
         Все инструкции должны использовать  только  относительную  адресацию
   как для ссылок на коды, так и на данные.
         Рекомендуется  держать все локальные данные на стеке и адресоваться
   к ним  относительно стека.
          Использовать заявление ORG 100Н в начале программы.
         Защищать регистры данных сегмент (DS), Базовый указатель (BP),  сте-
   ковый сегмент (SS) и стековый указатель (SP).
