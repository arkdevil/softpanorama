        спользованием заявления CALL ABSILUTE  в  ваших  программах  на  Тур-
   бо-Бейсике возбуждается подпрограмма языка ассемблера, которая заранее уже
   загружена с помощью заявления BLOAD или представляет собой часть программ-
   ного пространства данных.
         Например:  DEF SEG =  $H 5000
                    Моя &$ = Подпр% =&H4000
                    CALL ABSOLUTE Моя $ =Подпр%
         Обеспечивает  возбуждение  процедуры на языке ассемблера на смещении
   4000 шестнадцатиричное (16354 десятичное) в сегмегнте 5000 шестнадцатирич-
   ное (20480 десятичное) В более общем виде этот  адрес  специфициируется  в
   форме сегмент;смещение, как 5000:4000
         Система заявления CALL ABSOLUTE:
         CALL  ABSOLUTE [числовой идентификатор] где числовой идентификатор -
   это числовая скалярная переменная, оцениваемая в диапазоне  от  -32765  до
   65535.  Если идентификатор отрицателен, он принудительно переводится в по-
   ложительное значение с помощью арифметики двоичного дополнения. Это смеще-
   ние используется в сочетании с самым последним заявлением DEF SEG для  об-
   разования доступного вызову 32-битового абсолютного адреса. Если вы желае-
   те  использовать в качестве адреса какое-то выражение, то должны назначить
   его числовому идентификатору, а затем этот  идентификатор  использовать  в
   заявлении CALL.
         Опциональные  параметры ограничиваются толькь скалярными переменными
   в представлении коротких целых. Параметры подаются  посредством  ссылки  с
   помощью  16-битовых  указателей, проталкиваемых на стек в порядке слева на
   право. Все параметры располагаются в главном сегменте данных,  на  который
   указывает данных Сегмент (DS).
       Этот способ использования заявления CALL позволяет длбиться максималь-
   ной  совместимости  с интерпретаторным Бейсиком. Турбо-Бейсик обеспечивает
   более мощные SUB INLINE-процедуры, позволяющие получать  полный  доступ  к
   его возможностям расширенного хранения данных.
         После завершения выполнения желаемой функции процедурой на языке ас-
   семблера должен быть выполнен какой-либо внутрисегментный RETURN (возврат)
   к  Турбо-Бейсику.  Далее исполнение продолжается с заявления следующего за
   CALL.
         Программист  ответственен  за защиту регистра данных сегмента (DS) и
   стекового указателя (SP).
